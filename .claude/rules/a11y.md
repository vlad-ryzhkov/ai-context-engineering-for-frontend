# Accessibility (WCAG 2.2) Rules

- Interactive elements: use semantic HTML (`<button>`, `<a>`, `<input>`), NOT `<div onClick>`
- If `<div>` must be interactive: requires `role`, `tabIndex`, `onKeyDown` (Enter/Space), `aria-label`
- Images: `<img>` must have `alt` attribute (empty `alt=""` for decorative only)
- Touch targets: minimum 44x44px (WCAG 2.5.8)
- Color contrast: minimum 4.5:1 for normal text, 3:1 for large text (WCAG 1.4.3)
- Focus management: visible focus indicator, logical tab order
- Form fields: must have associated `<label>` or `aria-label`/`aria-labelledby`
- Live regions: use `aria-live="polite"` for dynamic content updates
